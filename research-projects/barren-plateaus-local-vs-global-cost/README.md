# Barren Plateaus: Does a Local Cost Function Actually Escape the Exponential Concentration of a Global One?

**Status:** complete, autonomously executed, reproducible end-to-end.

## Research question

Variational quantum algorithms (VQAs) train a parameterized quantum circuit
`U(theta)` to minimize a cost function `C(theta) = <psi(theta)| O |psi(theta)>`
using a classical optimizer. McClean et al. (2018) showed that for
sufficiently expressive/deep random circuits, the *variance* of
`dC/dtheta` for any parameter vanishes exponentially in the number of
qubits `n` — a "barren plateau" that makes gradient-based training
information-theoretically hard to escape from a random start, regardless
of the optimizer. Cerezo et al. (2021) then showed a refinement: if the
observable `O` is **local** (a sum of terms each acting on only a few
qubits) rather than **global** (a single operator acting on all `n`
qubits at once), *shallow* circuits can avoid the plateau entirely, because
the reduced state seen by a local observable only depends on gates inside
its causal light cone.

This project asks a quantitative version of that qualitative claim:

> **At a fixed, shallow circuit depth, how much shallower is the local
> cost's exponential concentration rate than the global cost's — and does
> that gap close as depth increases, or does the local cost settle onto
> its own, permanently shallower, exponential rate?**

This sits at the intersection of quantum computing, linear algebra, and
statistics (model-comparison fitting under measurement noise), and is
fully self-contained: no quantum hardware, no external data, no human
judgement calls beyond picking the ansatz — just an exact classical
statevector simulation, an exact analytic gradient rule, and a numerically
run sweep whose outcome was not fixed in advance. (An earlier version of
this ansatz contained a genuine bug — see "A wrong result and how it was
caught," below — which is itself part of the record.)

## Methodology

### 1. Circuit and cost functions

- **Ansatz:** `depth` layers, each applying `RY(theta)` then `RZ(phi)` to
  every qubit (2 random parameters/qubit/layer, drawn i.i.d.
  `Uniform(0, 2*pi)`), followed by a **linear** nearest-neighbor entangler
  `CNOT(0,1), CNOT(1,2), ..., CNOT(n-2,n-1)`. This is the standard
  "hardware-efficient ansatz" used throughout the barren-plateau
  literature. (`src/barren_plateaus/circuits.py`)
- **Global cost:** `C_G(theta) = 1 - |<0...0|psi(theta)>|^2` — the
  observable is the projector onto `|0...0>`, which acts on all `n` qubits
  jointly.
- **Local cost:** `C_L(theta) = 1 - (1/n) * sum_i <Z_i>(theta)` (rescaled
  to `[0,1]`) — a *sum* of `n` single-qubit observables, the textbook
  "local cost" from Cerezo et al. 2021 (not to be confused with reading
  out a single fixed qubit — see the bug writeup below for why that
  distinction matters).

### 2. Simulator

`src/barren_plateaus/statevector.py` is a from-scratch statevector
simulator: an `n`-qubit state is a complex tensor of shape `(2,) * n`, and
single-/two-qubit gates are applied as local tensor contractions
(`np.tensordot` + `moveaxis` for single-qubit gates, a transpose-reshape
trick for CNOT) rather than ever building the full `2^n x 2^n` unitary.
This is standard practice for classically simulating circuits up to a few
tens of qubits and keeps the `n=14`, `depth=16` sweep cell to well under a
second per sample.

### 3. Exact gradients via the parameter-shift rule

`RY` and `RZ` are both generated by an operator with eigenvalues `+-1/2`,
so for any cost function built from expectation values of such a circuit,

```
dC/dtheta_k = [C(theta + pi/2 * e_k) - C(theta - pi/2 * e_k)] / 2
```

is an **exact** identity (Mitarai et al. 2018; Schuld et al. 2019), not a
finite-difference approximation. `tests/test_gradients.py` cross-checks it
against central finite differences and against a closed-form single-qubit
case. The gradient is always probed at the middle qubit of the first
layer (`n // 2`) — see the bug section for why an edge qubit is a bad
choice.

### 4. Experimental sweep and model comparison

For every `(cost_type, depth, n)` in `{global, local} x {1,2,4,8,16} x
{2,4,6,8,10,12,14}`, 250 independent random parameter draws are sampled,
each contributing one exact gradient sample; the sample variance across
the 250 draws estimates `Var[dC/dtheta]` (`src/barren_plateaus/experiment.py`).
For each `(cost_type, depth)`, two competing models are fit by ordinary
least squares in log space across the 7 qubit counts
(`src/barren_plateaus/theory.py`):

- **exponential** (barren plateau): `log2(Var) = a - b*n`
- **power law** (no plateau): `log2(Var) = a - b*log2(n)`

and compared by `R^2`. A true barren plateau should be won decisively by
the exponential model; escaping one should favor the power law, or at
least a much smaller effective exponential rate `b`.

## Results

Full numeric output: `results/results.csv` (all 70 sweep cells) and
`results/summary.json` (all 10 model fits). Headline fitted exponential
rates `b` (`Var ~ 2^{-b*n}`), with `R^2` of the exponential fit in
parentheses:

| depth | global cost `b` | local cost `b` |
|------:|-----------------:|----------------:|
| 1     | 1.49  (R²=0.998) | 0.85  (R²=0.987) |
| 2     | 1.71  (R²=0.997) | 0.92  (R²=0.982) |
| 4     | 1.81  (R²=0.999) | 0.96  (R²=0.985) |
| 8     | 1.97  (R²=1.000) | 0.97  (R²=0.977) |
| 16    | 1.96  (R²=1.000) | 1.17  (R²=0.993) |

`results/variance_vs_n.png` plots `Var[dC/dtheta]` vs. `n` for every depth
and both cost functions side by side. `results/fit_slopes_vs_depth.png`
plots the fitted rate `b` and the exponential-model `R^2` as functions of
depth. `results/model_comparison_depth1.png` and `..._depth16.png` overlay
the raw data and fitted curves for the shallowest and deepest circuits
tested.

**Findings:**

1. **Both cost functions concentrate exponentially at every depth tested**
   — the exponential model wins (or ties) the `R^2` comparison in 9 of the
   10 `(cost_type, depth)` cells (all except local cost at depth 8, where
   the power law edges it out, R²=0.985 vs 0.977 — a genuinely close call
   worth flagging rather than rounding away). This is a stricter result
   than the purely qualitative "shallow local circuits have *no* barren
   plateau" reading of Cerezo et al. — for this ansatz and this specific
   local-cost definition, the plateau is present even at depth 1, just
   much shallower.

2. **The global cost's concentration rate roughly doubles the local
   cost's, and that gap is stable across depth.** Fitted `b_global` climbs
   from 1.49 (depth 1) to ~1.97 (depth 8-16, essentially the
   `Var ~ 2^{-2n} = 4^{-n}` rate expected for global cost functions with
   near-maximal entanglement). Fitted `b_local` climbs much more slowly,
   from 0.85 (depth 1) to 0.97 (depth 8), consistent with `Var ~ 2^{-n}`
   — a factor-of-two-slower exponential rate, not a qualitatively
   different scaling law. In other words: at every depth tested, the
   local cost buys you roughly *one extra qubit's worth of headroom per
   qubit added* compared to the global cost, but it does **not** buy you
   out of the exponential regime altogether — the gap does not visibly
   close (b_local stays well below b_global through depth 16), but it
   does not obviously *widen* either. Confirming whether that gap is
   truly depth-independent in the `n -> infinity` limit, versus slowly
   closing on a longer depth scale than tested here, would need larger
   `n` and depth than is tractable with exact statevector simulation.

3. **A late uptick at depth 16** (`b_local` jumps from 0.97 to 1.17)
   hints that the local cost's rate is *not* perfectly depth-independent
   — deeper circuits could be creeping the local cost's concentration
   rate toward the global one, consistent with the general expectation
   that sufficiently deep circuits eventually scramble local information
   as thoroughly as global information. The trend across depths
   1→2→4→8→16 is not perfectly monotonic (a small dip at depth 8 before
   the depth-16 uptick), which is expected sampling noise given 250 draws
   per cell — see `variance_stderr` in `results.csv` for per-cell error
   bars.

## A wrong result and how it was caught

The first full run of this sweep produced a nonsensical local-cost curve:
depth-1 local-cost gradient variance was pinned at machine-epsilon
(~1e-33) for *every* qubit count, wildly inconsistent with a "shallower
but still real" plateau. Tracing it down: the original ansatz used a
**ring** entangler (`CNOT(0,1), ..., CNOT(n-2,n-1), CNOT(n-1,0)`, wrapping
back to qubit 0), the gradient was probed at qubit 0's own rotation, and
the local cost read out qubit 0's `<Z>` alone. Using the CNOT conjugation
identity `CNOT_{c->t}^dagger Z_t CNOT_{c->t} = Z_t * Z_c`, conjugating
`Z_0` backward through the closed loop of the ring returns a `Z_0^2 = I`
factor — an exact, provable cancellation that makes `<Z_0>` after one
layer **mathematically independent of qubit 0's own rotation angle**,
regardless of `n`. That is not a physical result about circuit
trainability; it is an artifact of measuring a closed-loop topology at
exactly the qubit whose own parameter was being differentiated. Switching
to a linear (non-wrapping) entangler, averaging the local cost over all
qubits rather than reading out a single one, and probing the gradient at
the *middle* qubit of the first layer (`tests/test_circuits.py::test_linear_entangler_has_no_ring_degeneracy`
is a regression test for exactly this) fixed it; the numbers above are
from the corrected run. This is left in the README rather than quietly
fixed because catching your own instrumentation from producing physics
nonsense — instead of writing up "interesting" garbage — is itself part
of doing the science.

## Reproducing

```bash
cd research-projects/barren-plateaus-local-vs-global-cost
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

pytest tests/ -v                              # 36 tests, < 2s

python -m barren_plateaus.run_experiment      # full sweep, ~3 min
# regenerates results/results.csv, results/summary.json, and all PNGs
```
